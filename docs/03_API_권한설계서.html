원칙 · 역할
Principles
설계 원칙 · 확정 사항
결제 연동카카오페이 직접 연동(PG 미사용). 수수료 부담으로 규모 확대 시 PG 전환 대비 → 결제 어댑터로 추상화(PaymentProvider 인터페이스).
결제 단위차수(Round)가 최소 거래 단위. 신청·결제·취소·정산·환불 모두 차수별. 모임/동아리는 집계.
환불정산 잔액·취소분은 카카오페이 즉시 환불(account) 또는 장부 포인트 적립(ledger, 다음 모임 자동 차감) 중 선택.
신뢰/매너 산식점수·매너온도·등급 계산식은 서버 내부 로직. API는 결과 필드만 노출(GET /me/trust, /users/{id}).
본인인증 게이트금융 행위(납부·개설·정산·환불·연동)는 verified 필수. 열람·소셜은 auth.
감사·정합성장부 append-only, 결제 webhook 멱등, 민감 작업 AuditLog.

역할 모델 (RBAC)

3계층 — ① 플랫폼 역할 ② 운영자 세부 역할 ③ 동아리 역할(리소스 스코프). 권한은 토큰의 클레임 + 리소스 멤버십으로 평가.

결제 플로우 · 커버리지
KakaoPay Flow
카카오페이 결제 · 환불 플로우

PG 없이 카카오페이를 직접 호출한다. 결제 로직은 PaymentProvider 어댑터 뒤에 두어 추후 PG 전환 시 교체만으로 대응한다.

1. 신청유저가 모임 상세에서 차수 선택 → POST /attendances (roundIds)
2. 결제 준비POST /payments/kakao/ready — 차수 합산액으로 카카오페이 결제 준비(tid 발급)
3. 사용자 승인카카오페이 앱/딥링크에서 결제 승인
4. 승인 처리POST /payments/kakao/approve(콜백) + webhook(멱등) → 결제 완료, 차수별 Attendance=paid, 장부 분개
5. 모임통장 수취동아리 정기 모임은 모임통장으로 자동 수취(독촉 불필요)
6. 정산·환불정산 마감 → 잔액 1인당 환급: 카카오페이 즉시 환불(account) 또는 장부 포인트 적립(ledger)
멱등·정합성 · approve 콜백과 webhook은 같은 결제를 두 번 확정하지 않도록 partner_order_id 기준 멱등. 환불도 동일.
