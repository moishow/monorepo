기간 제한형 에스크로 흐름

단계별 상세

1

모집 · 예치

OPEN

부원이 모임 차수별 예상 금액을 앱 포인트로 예치. 만남 시간 전까지 자유 취소 → 즉시 본인 지갑 환불. 총무 출금 불가.

  • 부원: 차수 선택 → 포인트 예치(부족분 카카오페이 충전)
  • 부원: 취소 → 예치 포인트 즉시 환불
  • 총무: 출금 권한 ❌ (잠김)
2

만남 시간 마감 (락)

LOCKED

약속 시간 도달 순간 부원 취소 자동 차단. 동시에 총무에게만 해당 모임 자금 출금(현금화) 권한 활성화.

  • 시스템: 마감 시각 도달 → 취소 API 차단
  • 시스템: 총무 출금 권한 활성화
  • 총무→부원: 락 스냅샷 전원 출금 동의(CONSENT) 요청 → 전원 동의 시에만 출금 개방
  • 동시성 격리: 마감 직전 취소/출금 요청 순차 처리
3

총무 출금 · 지출

WITHDRAWN

총무가 모인 자금을 실제 본인 계좌로 인출해 오프라인 비용으로 사용. 동시에 정산 데드라인 타이머 작동.

  • 총무: 모임 자금 → 본인 계좌 출금(현금화)
  • 시스템: 정산 데드라인 타이머 시작(기본 12h, FeePolicy 설정 가능)
  • 광고: 출금 처리 대기(3~5초) 화면 노출
4

증빙 · 잔액 반납

SETTLING

모임 종료 후 총무가 영수증 사진 + 실제 지출액 입력, 출금액−지출액 = 남은 잔액 포인트를 시스템에 즉시 반납. 지연 시 패널티.

  • 총무: 영수증 업로드 → OCR 자동 추출(금액·가게·일시)
  • 총무: 잔액 포인트 시스템 반납
  • 시스템: 데드라인 초과 시 신용등급 차감
5

자동 정산 · 현금화

DONE

시스템이 반납금 ÷ 참여인원 계산해 부원에게 공평 환급. 부원은 포인트를 원할 때 즉시 본인 계좌로 현금화.

  • 시스템: 반납금 / 참여인원 → 1인당 환급
  • 부원: 환급 포인트 → 즉시 계좌 현금화(출금)
  • 광고: 정산 피드 확인 화면 노출
2
상태 머신
2 · State Machines
상태 전이

만남 시각(LOCK)과 정산 데드라인(타이머)이 상태 전이의 두 트리거다. 모든 전이는 AuditLog + 원장 분개로 기록된다.

Meeting / Round Escrow (자금 락 사이클)

OPEN→LOCKED→CONSENT→WITHDRAWN→SETTLING→DONE

OPEN(예치·취소 자유, 출금 잠김) → 만남시각 도달 → LOCKED(취소 차단, 출금 활성) → 락 스냅샷 전원 출금 동의 → CONSENT(전원 동의 시 출금 개방) → 총무 출금 → WITHDRAWN(정산 타이머) → 영수증·잔액 반납 → SETTLING → 자동 균등 정산 → DONE. (openapi Escrow.state 정본)

Deposit (부원 예치)

deposited→refunded(취소)→locked→settled-refund

예치 → (마감 전) 취소 시 refunded → (마감 후) locked → 정산 후 남은 분 settled-refund로 지갑 환급.

TreasurerPayout (총무 출금)

blocked→enabled→withdrawn→overdue→penalized

마감 전 blocked → 마감 후 enabled → 출금 withdrawn(타이머 시작) → 데드라인 초과 overdue → 미반납 지속 시 penalized.

Settlement (증빙·반납)

open→withdrawn→settling→returned→done

open → 총무 출금 withdrawn → 영수증 OCR 검증·반납 진행 settling → 잔액 반납 완료 returned → 시스템 균등 분배 done. (openapi Settlement.status 정본; 내부 단계로 receipt-verified·auto-settled를 둠)

⏱ 두 개의 시계 · ① 만남 시각 도달 → OPEN→LOCKED(취소 차단·출금 활성). ② 정산 데드라인 타이머(총무 출금 시점부터, 기본 12h·FeePolicy 설정 가능) 초과 → 신용등급 패널티. 둘 다 서버 스케줄러가 트리거.
3
포인트 원장
3 · Point Ledger
포인트 원장 모델

모든 포인트 이동은 append-only 원장(LedgerEntry)으로 분개된다. 회원 간 임의 송금은 존재하지 않으며, 흐름은 충전 → 예치 → (환불/락) → 총무출금 → 반납 → 정산환급 → 현금화로 고정된다.

type부호설명
CHARGE+카카오페이로 포인트 충전(부족분). 수수료 0원
DEPOSIT−부원 지갑 → 모임 에스크로 예치(차수 단위)
REFUND_CANCEL+마감 전 취소 → 예치 포인트 즉시 환불
LOCK=마감 시 예치금 락(이동 없음, 상태 전이만)
PAYOUT_TREASURER−에스크로 → 총무 계좌 출금(현금화)
RETURN+총무가 미사용 잔액 포인트 시스템 반납
SETTLE_REFUND+반납금 ÷ 참여인원 → 부원 지갑 균등 환급
CASHOUT−부원 지갑 포인트 → 본인 계좌 현금화
PENALTY=반납 지연 시 신용등급 차감(금액 이동 없음, 점수 기록)
✅ 정합성 불변식 · 모임 에스크로 잔액 = Σ DEPOSIT − Σ PAYOUT_TREASURER + Σ RETURN − Σ SETTLE_REFUND. 정산 완료(DONE) 시 에스크로 잔액 = 0, 시스템 발행 포인트 최소화.

지갑 · 계좌

Wallet (부원)포인트 잔액 + 거래내역. 충전(카카오페이)·현금화(계좌)만 외부 연결. 송금 기능 없음.
MeetingEscrow모임/차수 단위 에스크로. 예치금 집계 + 락 상태 + 총무 출금/반납 추적.
BankAccount현금화(출금) 대상 계좌. 부원=환급 수취, 총무=모임 자금 인출.

정책 · 예외 처리
4 · Operating Policy
신용 등급 · 패널티

총무의 먹튀·지연을 시스템이 자동 제어한다. TrustProfile(매너/신뢰 점수)과 연동.

반납 데드라인모임 종료 후 지정 기한(기본 12시간, FeePolicy 설정 가능) 내 영수증 증빙 + 잔액 반납 완료
지연 차감데드라인 초과 시 신용등급 점수 자동 차감(초과 시간 비례 가능)
권한 박탈등급이 임계 이하 → 향후 총무(방장) 권한 박탈, 신규 모임 개설/출금 제한
회복정상 정산 누적 시 점수 회복. TrustProfile.score/grade와 연동

영수증 조작 방지 (OCR)

추출 항목결제 금액 · 가게명 · 결제 일시를 영수증 이미지에서 자동 추출
강제 매핑추출 금액이 지출 입력창에 강제 매핑 → 총무 임의 금액 위조 차단
검증추출 금액 ≠ 입력 금액 시 경고/반려. 다중 영수증 합산 지원
저장원본 이미지 + OCR 메타(금액·상호·일시·신뢰도) 보관(감사 추적)
강제 매핑 · OCR 추출 금액이 지출 입력값으로 강제 매핑되어 총무의 임의 금액 위조를 원천 차단. 불일치 시 반려.

동시성 · 인터셉트 방어

마감 경합만남 기한 직전 대량 동시 취소 → 모임방 상태 변화 요청 격리·순차 처리
취소 vs 출금총무 출금 타이밍과 부원 취소 타이밍 충돌 방지 → 모임 단위 직렬화 락
멱등출금·반납·환급은 Idempotency-Key로 중복 방지. 원장 append-only
정합성에스크로 잔액 = Σ(DEPOSIT) − Σ(PAYOUT_TREASURER) + Σ(RETURN) − Σ(SETTLE_REFUND) 항상 검증

법적 전략 · API
5 · Legal & Business
법적 규제 · 방어 전략
선불업 등록 유예포인트는 외부 범용 사용 불가, 오직 특정 모임방 정산 목적. 정산 후 즉시 현금 인출 → 발행 잔액 최소화로 선불업 등록 면제 기준 유지. 회원 간 임의 송금 완전 제한.
수수료 0원충전·출금 수수료 미수취 → 세법상 비영리 플랫폼 지위. 초기 런칭 시 사업자 등록 없이 합법 운영 가능.
광고 수익자금 결제선에서 수수료 대신 인앱 광고(AdMob 등). 총무 출금 대기(3~5초)·정산 피드 화면에 배치. 금융 규제 무관한 정당 수익, 소액은 종합소득세 신고로 처리.
⚖️ 설계 제약(법적 근거) · ① 회원 간 임의 송금 API 제공 금지 ② 포인트는 모임 정산 목적 외 사용 불가 ③ 충전·출금 수수료 0원 ④ 정산 완료 즉시 현금화 유도(발행 잔액 최소화). 이 4가지는 선불업 등록 유예의 전제이므로 구현 시 반드시 유지.

API 엔드포인트 (에스크로)

기존 API에 추가/대체되는 에스크로 전용 엔드포인트. 권한은 단계별로 제한된다.

POST/meetings/{id}/rounds/{rid}/deposit — 부원 차수 예치(포인트 차감, 부족분 충전) [verified]
DELETE/meetings/{id}/rounds/{rid}/deposit — 마감 전 취소 → 즉시 환불 (LOCKED면 409) [self]
POST/meetings/{id}/lock — 만남 시각 도달 → 락(스케줄러/시스템) [system]
GET/meetings/{id}/escrow — 에스크로 잔액·락 상태·참여 인원 [club.member]
POST/meetings/{id}/payout — 총무 출금(현금화) — LOCKED 이후만, 타이머 시작 [club.treasurer]
POST/meetings/{id}/settlement/receipts — 영수증 업로드 → OCR 추출 [club.treasurer]
POST/meetings/{id}/settlement/return — 실지출 입력 + 잔액 포인트 반납 [club.treasurer]
POST/meetings/{id}/settlement/auto — 반납금 ÷ 인원 자동 균등 환급(시스템) [system]
POST/me/wallet/charge — 카카오페이 포인트 충전(수수료 0) [verified]
POST/me/wallet/cashout — 포인트 → 본인 계좌 현금화(수수료 0) [self]
GET/me/wallet — 포인트 잔액·거래내역 [self]
✅ 대체 관계 · 기존 "참석 신청 = 카카오페이 즉시 이체"는 "예치 → 락 → 총무 출금" 모델로 대체된다. 카카오페이는 이제 포인트 충전·현금화 레일로만 쓰인다. OpenAPI에 동일 반영.
