Escrow Sequence
포인트 에스크로 예치·출금·정산 시퀀스

만남 시각(LOCK)과 정산 데드라인(타이머)이 상태 전이의 두 트리거. 모든 포인트 이동은 append-only 원장에 분개.

1. 예치POST /meetings/{id}/rounds/{rid}/deposit (Idempotency-Key) → deposited (OPEN)
2. 락만남 시각 → POST /meetings/{id}/lock (스케줄러) → LOCKED, 취소 차단
3. 동의payout-consent/request → 락 스냅샷 전원 vote(agree) → allAgreed (CONSENT)
4. 출금POST /meetings/{id}/payout → 총무 계좌 현금화 (WITHDRAWN), 데드라인 타이머 시작
5. 증빙·반납settlement/receipts(OCR) → settlement/return(실지출+잔액 반납) (SETTLING)
6. 자동 정산settlement/auto → 반납금 ÷ 인원 균등 환급 (DONE) → 부원 cashout
실패·복구 · 충전 webhook 미수신 시 tid로 주문 조회 폴링 폴백. 충전·예치·출금·반납·환급은 Idempotency-Key로 멱등, 원장 append-only.

상태 전이 요약

EscrowOPEN → LOCKED → CONSENT → WITHDRAWN → SETTLING → DONE
Depositdeposited → (취소) refunded / (락) locked → settled_refund
TreasurerPayoutblocked → enabled → withdrawn → (초과) overdue → penalized
Settlementopen → withdrawn → settling → returned → done
차수 단위 원칙 · 한 모임에 여러 차수를 신청해도 예치·취소·락·반납·정산·환급은 모두 차수별 Deposit·LedgerEntry로 분개되어 부분 취소·노쇼·부분 환불이 독립 처리된다. 회원 간 송금 API 없음.
